# 翻译审校技能

## 触发方式

当用户请求审校翻译文件时使用此技能。常见触发语：

- "审校翻译"
- "检查这个翻译"
- "对照原文看看翻译有没有问题"

## 审校流程

### 第 1 步：准备工作

- 打开英文原文和中文翻译（通常为 `{页码范围}带翻译.md`）
- 确认文件格式是否符合工作区 Markdown 规范

### 第 2 步：逐段对照

对每一段执行以下检查：

1. **完整性检查** — 是否有漏译（整段或整句遗漏）
2. **准确性检查** — 译文是否忠实原文含义
3. **术语一致性** — 同一术语在全篇中是否翻译一致
4. **人名/地名** — 是否使用通行译名
5. **语序与流畅度** — 中文表述是否自然

### 第 3 步：问题清单

将发现的问题整理为结构化清单：

```markdown
| # | 位置 | 问题类型 | 原文 | 当前译文 | 建议修改 |
|---|---|---|---|---|---|
| 1 | § 3 | 误译 | ... | ... | ... |
```

### 第 4 步：修正建议

- 对每个问题给出具体修改建议
- 标注置信程度（确定 / 建议 / 讨论）

### 第 5 步：总结

- 统计问题数量和类型分布
- 给出整体翻译质量评价

## 问题类型分类

| 代码 | 类型 | 说明 |
|---|---|---|
| `漏译` | 遗漏翻译 | 原文内容在译文中缺失 |
| `误译` | 翻译错误 | 译文含义偏离原文 |
| `术语` | 术语不一致 | 同一词/短语前后翻译不同 |
| `表达` | 表达不当 | 中文不自然、生硬或冗余 |
| `格式` | 格式问题 | 排版、标点、斜体等格式错误 |

## 输出模板

```markdown
# {文件名} — 翻译审校报告

## 审校概要
- **原文文件**：...
- **译文文件**：...
- **审校日期**：...

## 问题清单
| # | 位置 | 类型 | 原文 | 当前译文 | 建议修改 | 置信度 |
|---|---|---|---|---|---|---|

## 总结
- 发现问题 X 处：漏译 X、误译 X、术语 X、表达 X、格式 X
- 整体评价：...
```
