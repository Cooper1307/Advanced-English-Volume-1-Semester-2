# 高级英语（第一册）课程分析工作区 — 规则

## 工作区身份

本工作区用于「高级英语（第一册）」课程内容的 AI 辅助分析，包括课文精读、翻译审校、词汇整理等任务。教材按页码范围划分课文。

## 语言规则

- **分析输出**：默认使用**中文**撰写。
- **原文引用**：保留英文原文，使用 `>` 引用块或行内代码标注。
- **术语标注**：首次出现的专业术语需附英文原文，如：修辞手法（rhetorical device）。

### 归档流程

- 原始 `.txt` 文件在整理为 `.md` 后移入 `归档/` 文件夹。
- `归档/` 中的文件仅作留底，不再编辑。

### AI 文件排除规则

以下目录和文件 **不应主动读取**，以避免浪费 token：

- `归档/` — 仅存放已整理的原始 `.txt` 文件留底，内容已迁移至对应 `.md` 文件
- `.agents/` — 仅在需要查阅规则或技能定义时读取
- 任何 `.tmp`、`.bak` 后缀的临时文件

> **原则**：优先读取课文子目录下的 `.md` 文件（如 `Lesson09_*/`），避免重复读取归档中的同名 `.txt` 文件。

### Markdown 格式标准

- 标题层级：`#` 课名 → `##` 大板块 → `###` 子板块
- 有序列表使用 `1.` `2.` 标准格式（不用 `1)` `2)`）
- 注释使用 `> **[n]**` 引用块格式
- 书名、外来语斜体：`*Animal Farm*`、`*in terrorem*`

## 课文分析标准框架

对课文进行分析时，应覆盖以下维度（可按需增减）：

1. **背景与主题** — 作者介绍、写作背景、文章主旨
2. **篇章结构** — 段落划分、逻辑关系、行文线索
3. **词汇与短语** — 重点词汇、固定搭配、同义/反义词
4. **句法分析** — 长难句拆解、语法要点、句式特征
5. **修辞手法** — 比喻、讽刺、排比、象征等
6. **翻译难点** — 文化负载词、双关语、特殊句式的翻译策略
7. **思考题与讨论** — 基于 Guiding Questions 的深度解读

### 词汇深度分析要求

对重点词汇进行分析时，除基本词义外，还应包括：

- **词根与词缀** — 拆解词根（root）、前缀（prefix）、后缀（suffix），追溯拉丁/希腊语源（如 *imperialism* ← *imperium*（最高权力）+ *-ism*）
- **词源学分析（Etymology）** — 该词从古英语 / 拉丁语 / 法语等来源到现代英语的演变路径
- **含义演变逻辑** — 不同历史阶段中词义的拓展、缩小、转移或褒贬义变化
- **母语者直觉** — 该词在英语母语者脑海中唤起的画面、情绪或联想（connotation），而不仅仅是字典释义（denotation）
- **易混辨析** — 与近义词、形近词的用法区别

### 解析时效性与信息增强规则

- **时间校对** — 每次执行分析任务时，必须先校对当前北京时间（CST, UTC+8），并在输出中标注解析时间戳。
- **联网增强** — AI 应自行判断是否需要联网搜索以增强解析的准确性和时效性。以下场景应优先联网：
  - 验证作者生卒年、历史事件等事实性信息
  - 查证术语的最新学术释义或权威用法
  - 核实文化背景或跨文化翻译中的争议点
- **引用规范** — 联网获取的信息必须引用真实存在的来源（书名、网站、论文等），不得编造引用。

## 翻译规范

- 中英对照排版时，英文原段在前，中文翻译紧跟其后。
- 人名、地名首次出现时附注中文译名。
- 同一篇分析中术语翻译必须前后一致。
